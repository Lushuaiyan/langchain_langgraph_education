# agent的高级运用

## 1. 多轮对话
LLM都是无状态的, 因此需要多轮对话时, 需要开发者将历史消息全部作为消息发送给LLM

在langchain中, agent是有状态的, messages就是它维持的一个状态, 所以当我们需要使用多轮对话时, 只需将更新的信息添加到messages中即可


## 2. 参数模型
在定义工具时, 常常需要使用的参数是一个复杂参数, 需要使用`pydantic`定义
`@tool`装饰器除了会将`docstring`传递给模型之外, 还会将参数Schema(包含默认值和描述)传递给模型
所以只要定义工具时对参数进行类型注解, 模型就可以基本使用这个工具
- `docstring`告诉agent什么时候需要使用该工具
- 类型注解告诉agent如何使用该工具
